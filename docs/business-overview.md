# 业务概览（Profession / Roadmap / Course / Node / Post / 用户目录树）

最后更新：2025-09-22

## 目标与范围
- 面向阅读与理解，统一本项目中的核心业务概念与它们的关系。
- 聚焦“职业、路线图、课程、节点、帖子、用户目录树”的语义、边界与关键流程。
- 不涉及具体实现细节与代码接口；如需 API 规格，请另见后续接口文档。

## 核心概念
- 职业（Profession）：行业/岗位维度的学习领域。
- 路线图（Roadmap）：归属某个职业的学习路径，表现为图（nodes/edges/JSON）；可开始学习、点赞、置顶等。
- 课程（Course）：学习单元；创建课程时系统会自动生成一个默认根节点；课程可有父子关系（子课程）。
- 节点（Node）：构成课程树或用户目录树的基本结构单元；可标记完成。
- 帖子（Post）：挂载在节点下的内容单元，分两类：
  - 目录型：承载一棵“目录模板子树”，供他人采纳。目录型帖子一经创建不可更新（结构冻结）。
  - 文章型：节点的具体内容，可编辑且可置顶。
- 用户目录树（User Directory Trees）：
  - 每个用户在“每一门课程”下，都维护一个“目录树数组”，用于个性化组织学习结构。
  - 默认读取数组的第一个目录树；用户可手动切换到其他目录树。

## 实体关系（概览）
- Profession 1 — * Roadmap（路线图隶属职业）。
- Course 拥有系统层面的根节点与默认结构；同时，“每位用户对此课程”拥有“多个私有目录树（数组）”。
- Node 可属于课程系统树，也可属于某个用户目录树（私有）。阅读与进度围绕“用户目录树”展开。
- Post 隶属某个节点：
  - 文章型 Post：节点内容的承载。
  - 目录型 Post：保存一棵目录模板子树，供他人采纳到自己的某个目录树中。

## 关键业务规则（不变式）
- 目录型帖子不可更新：创建后结构冻结，后续仅可被采纳，不可修改模板。
- 仅能修改自己的目录树：用户对自己的目录树与其节点拥有增删改移动权限，不能修改他人的目录树。
- 多目录树与默认：每用户每课程持有“目录树数组”，默认读取索引0；用户可切换当前使用的目录树。
- 至少保留一棵目录树：不可全部删除为 0；系统需校验阻止该情况。
- 总进度口径固定：总是按目录树数组的“第一个”进行统计与展示。
- 采纳幂等并替换：同一目录树的同一路径被重复采纳时，后一次内容整体替换前一次内容（覆盖该处子树）。

## 目录型 Post 的生命周期
1) 创建：在某个节点下创建“目录型帖子”时，系统会生成并关联一棵“目录模板子树”。
2) 不可更新：目录型帖子一经创建不可更新（包括其目录结构）。
3) 被采纳：其他用户可选择该目录型帖子，将其模板子树克隆并插入到“自己的某棵目录树”的指定路径下。

## 采纳（Adopt）流程与规则
- 输入条件：课程上下文、目标目录树（数组中的某一棵）、当前阅读路径 `path`（如 `1-2-3`，1-based）。
- 操作行为：把目录型帖子的模板子树“克隆”为用户私有节点，并插入到该目录树的该 `path` 下。
- 替换策略：如果该 `path` 位置已存在子树，则以“后者替换前者”的方式整体覆盖（幂等、可重复执行）。
- 交互建议：因可能覆盖用户手动修改的内容，前端应进行二次确认提示；后端按替换规则执行。

## 阅读与路径（Path）
- 阅读始终基于“当前选中的目录树”，默认是目录数组的第一个。
- 系统使用 1-based 的路径字符串（例如 `1-2-3`）来标识当前位置与插入位置。
- 路径与排序：当用户对目录节点进行排序变更后，路径也会随之变化；历史路径链接的兼容性需按产品策略处理。

## 文章型 Post（编辑与置顶）
- 可编辑：支持修改文章内容。
- 可置顶：同一节点下可将文章置顶，置顶文章优先展示；与时间排序的组合策略由产品确定。

## 用户目录树管理
- 新建/切换：用户可在同一课程下创建多棵目录树，并在阅读时手动切换当前目录。
- 删除：允许删除非最后一棵目录树；不可删到 0。若删除当前默认（索引0），通常需要将下一棵前移为新的索引0（具体以实现为准）。
- 重排：改变数组顺序会影响默认目录（索引0），进度统计随之基于新的索引0进行。

## 进度与统计
- 完成度：节点完成标记绑定在“用户私有节点”（属于某棵用户目录树）。
- 课程总进度：按“目录树数组的第一个”计算（已完成节点数 / 总节点数）。
- 互动数据：Post 的点赞/评论/浏览量不随采纳复制；采纳复制的是目录结构，而非互动数据。

## 权限与安全
- 目录型 Post：创建者与他人均不可更新其模板结构；仅可采纳。
- 目录树：用户仅能操作自己在某课程下的目录树与其节点（增删改、采纳/替换、排序、完成标记等）。
- 管理审核：删除违规内容等走独立的管理流程，不改变“目录型 Post 不可更新”的原则。

## 边界与冲突处理
- 重复采纳：允许，后一次覆盖前一次（同一路径）。
- 手工修改冲突：若用户已手工修改该 `path` 下的子树，采纳会整体替换；需前端确认提示。
- 默认目录变更：当用户更改目录树数组顺序时，新的索引0成为默认目录，进度口径切换到新默认。
- 目录模板下架：目录型帖子被作者删除/下架时，已采纳到用户目录的克隆子树通常保持不受影响（推荐策略）。

## 已确认事项（补充）
1) 目录树数量上限：默认最多 10 棵，且为可配置项。
2) 删除默认目录：删除数组第一个目录后，后面的目录自动前移为新的第一个，并重新计算进度。
3) 采纳确认：需要交互确认（防止覆盖已有子树）。
4) 路径稳定性：无历史链接需求；目录仅用于用户本人查看，不需要链接兼容。
5) 置顶策略：同一节点允许多个置顶，按置顶时间排序显示。
6) 路线图绑定：路线图的节点即课程；一个课程树就是一个路线图。

## 开放问题（待进一步细化）
- 目录树数量上限的配置入口与作用域（系统级/用户级/课程级）需要界定。
- 删除默认目录后的“前移与进度重算”在前端是否需要显式提示用户。
- 置顶时间的定义（创建置顶时刻/最后置顶更新时间）与与普通排序的融合规则。
- 采纳时的覆盖确认文案及是否提供“撤销最近一次采纳”的快捷操作。
- 路线图与课程的一致性维护场景（例如课程结构大改时对应路线图视图是否联动刷新）。
