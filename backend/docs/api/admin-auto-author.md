# AutoAuthor 管理接口文档

> 控制器: `AdminAutoAuthorController`
> 基础路径: `/api/v1/admin/auto-author`
> 认证要求: 所有接口需要登录
> 限流策略: 30次/分钟（按用户）

---

## 功能概述

AutoAuthor 是一个自动内容生成系统，用于为课程节点自动生成帖子内容。本接口提供管理功能，包括：
- 扫描缺少内容的节点
- 手动将节点加入生成队列
- 重置 AI 会话
- 清空生成队列

## 1. 扫描节点

### 接口信息
- **路径**: `POST /api/v1/admin/auto-author/scan`
- **描述**: 扫描数据库中缺少自动生成内容的节点，并将其加入生成队列
- **认证**: 需要登录
- **权限**: 需要管理员权限（建议）

### 功能说明
- 扫描最多 10 页节点（每页数量由系统配置决定）
- 查找未删除且缺少指定用户帖子的节点
- 使用 Redis 游标记录扫描进度（key: `autoAuthor:scan:lastId`）
- 按 ID 递增顺序扫描，扫描到末尾后自动重置游标
- 将符合条件的节点加入 Redis 就绪队列

### 请求参数
无

### 响应参数

#### 成功响应 - 200 OK
```json
{
  "code": 200,
  "message": "操作成功",
  "timestamp": 1735459200000
}
```

### 使用示例

```bash
curl -X POST "http://localhost:8080/api/v1/admin/auto-author/scan" \
  -H "token: your-admin-token"
```

**响应**:
```json
{
  "code": 200,
  "message": "操作成功",
  "timestamp": 1735459200000
}
```

### 业务逻辑

1. **扫描流程**：
   - 从 Redis 读取上次扫描的最后一个节点 ID
   - 按 ID 递增顺序查询下一页节点
   - 过滤出未删除且缺少自动内容的节点
   - 将符合条件的节点 ID 加入 Redis 队列
   - 更新游标到当前页最后一个节点 ID

2. **终止条件**：
   - 扫描了 10 页
   - 或某一页返回 0 个节点（到达末尾）

3. **游标重置**：
   - 当扫描到数据库末尾（没有更多节点）时
   - 游标自动重置为 0，下次从头开始扫描

### 注意事项
- 此接口是手动触发扫描，系统也会定时自动扫描
- 扫描不会重复处理已在队列中的节点
- 扫描进度存储在 Redis 中，重启后保留
- 如果 AutoAuthor 功能被禁用，扫描仍会执行但不会生成内容

## 2. 加入生成队列

### 接口信息
- **路径**: `POST /api/v1/admin/auto-author/enqueue/{nodeId}`
- **描述**: 手动将指定节点加入 AutoAuthor 生成队列
- **认证**: 需要登录
- **权限**: 需要管理员权限（建议）

### 功能说明
- 手动触发为某个节点生成内容
- 检查 AutoAuthor 功能是否启用
- 将节点 ID 加入 Redis 就绪队列
- 后台任务会自动处理队列中的节点

### 请求参数

#### 路径参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| nodeId | Long | 是 | 节点ID，必须大于 0 |

### 响应参数

#### 成功响应 - 200 OK
```json
{
  "code": 200,
  "message": "操作成功",
  "timestamp": 1735459200000
}
```

#### 参数错误 - 200 OK (code: 1002)
```json
{
  "code": 1002,
  "message": "节点ID必须大于0",
  "timestamp": 1735459200000,
  "path": "/api/v1/admin/auto-author/enqueue/0"
}
```

### 使用示例

#### 示例 1: 成功加入队列
```bash
curl -X POST "http://localhost:8080/api/v1/admin/auto-author/enqueue/12345" \
  -H "token: your-admin-token"
```

**响应**:
```json
{
  "code": 200,
  "message": "操作成功",
  "timestamp": 1735459200000
}
```

#### 示例 2: 无效的节点ID
```bash
curl -X POST "http://localhost:8080/api/v1/admin/auto-author/enqueue/-1" \
  -H "token: your-admin-token"
```

**响应**:
```json
{
  "code": 1002,
  "message": "节点ID必须大于0",
  "timestamp": 1735459200000,
  "path": "/api/v1/admin/auto-author/enqueue/-1"
}
```

### 业务逻辑

1. **功能检查**：
   - 检查 `system.auto-author.enabled` 配置
   - 如果功能未启用，直接返回成功（不执行操作）

2. **入队操作**：
   - 将节点 ID 添加到 Redis 队列
   - 队列使用 Set 数据结构，自动去重
   - 后台定时任务会定期处理队列

3. **不验证节点是否存在**：
   - 接口不检查节点是否真实存在
   - 生成任务执行时会检查节点有效性

### 使用场景
- 新创建的节点需要立即生成内容
- 某个节点的内容生成失败，需要重试
- 手动触发特定节点的内容生成

### 注意事项
- 重复调用会去重，不会重复生成
- 节点必须未删除且符合生成条件
- 生成任务是异步执行的，不会立即完成
- 如果功能被禁用，调用接口会静默成功（但不执行）

## 3. 重置AI会话

### 接口信息
- **路径**: `POST /api/v1/admin/auto-author/session/reset`
- **描述**: 重置 AutoAuthor 的 AI 会话状态
- **认证**: 需要登录
- **权限**: 需要管理员权限（建议）

### 功能说明
- 清除当前的 AI 会话上下文
- 强制创建新的会话进行内容生成
- 用于解决会话异常或需要重新开始对话的情况

### 请求参数
无

### 响应参数

#### 成功响应 - 200 OK
```json
{
  "code": 200,
  "message": "操作成功",
  "timestamp": 1735459200000
}
```

### 使用示例

```bash
curl -X POST "http://localhost:8080/api/v1/admin/auto-author/session/reset" \
  -H "token: your-admin-token"
```

**响应**:
```json
{
  "code": 200,
  "message": "操作成功",
  "timestamp": 1735459200000
}
```

### 业务逻辑

1. **会话重置**：
   - 清除内存中的会话ID
   - 下次生成时会自动创建新会话

2. **影响范围**：
   - 只影响后续的内容生成
   - 不影响正在生成的任务
   - 不影响已生成的内容

### 使用场景
- AI 会话出现异常或错误
- 会话上下文太长影响生成质量
- 需要重新开始对话以获得更好的效果
- 测试或调试 AI 生成功能

### 注意事项
- 重置会话会丢失当前的对话上下文
- 建议在没有正在进行的生成任务时执行
- 会话会在下次生成时自动重新创建
- 不会影响 Redis 队列中的待处理节点

## 4. 清空生成队列

### 接口信息
- **路径**: `DELETE /api/v1/admin/auto-author/queue`
- **描述**: 清空 Redis 中的所有待处理节点队列
- **认证**: 需要登录
- **权限**: 需要管理员权限（建议）

### 功能说明
- 删除 Redis 就绪队列中的所有节点 ID
- 返回删除的节点数量
- 不影响正在生成中的任务

### 请求参数
无

### 响应参数

#### 成功响应（队列非空）- 200 OK
```json
{
  "code": 200,
  "message": "操作成功",
  "data": "已清空队列，共删除 15 个待处理节点",
  "timestamp": 1735459200000
}
```

#### 成功响应（队列已空）- 200 OK
```json
{
  "code": 200,
  "message": "操作成功",
  "data": "队列已经是空的",
  "timestamp": 1735459200000
}
```

### 使用示例

#### 示例 1: 清空非空队列
```bash
curl -X DELETE "http://localhost:8080/api/v1/admin/auto-author/queue" \
  -H "token: your-admin-token"
```

**响应**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": "已清空队列，共删除 15 个待处理节点",
  "timestamp": 1735459200000
}
```

#### 示例 2: 清空已空队列
```bash
curl -X DELETE "http://localhost:8080/api/v1/admin/auto-author/queue" \
  -H "token: your-admin-token"
```

**响应**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": "队列已经是空的",
  "timestamp": 1735459200000
}
```

### 业务逻辑

1. **清空操作**：
   - 删除 Redis 中存储队列的 Set
   - 统计删除的节点数量
   - 返回友好的提示消息

2. **不影响正在执行的任务**：
   - 只清空待处理队列
   - 正在生成中的任务不受影响

### 使用场景
- 停止所有待处理的生成任务
- 重新规划内容生成策略
- 测试或调试时清理队列
- 队列中积压了大量无效节点

### 注意事项
- **谨慎操作**：此操作不可逆
- 清空后需要重新扫描或手动入队才能恢复
- 不会删除已生成的内容
- 不会影响扫描进度游标

## 通用说明

### 认证与授权
- **登录验证**：所有接口都使用 `@SaCheckLogin` 要求用户登录
- **权限控制**：建议配合 `@RequireRole(ADMIN)` 限制为管理员使用
- **Token 传递**：通过 `token` header 传递认证令牌

### 限流策略
- **限流规则**：30次/分钟/用户
- **限流类型**：按用户 ID 限流（`LimitType.USER`）
- **超出限制**：返回 429 或限流错误码

### 错误码

| 错误码 | 说明 | 场景 |
|--------|------|------|
| 200 | 成功 | 操作成功 |
| 1002 | 参数异常 | 节点ID无效 |
| 1101 | 用户未登录 | 未提供token或token无效 |
| 2301 | 访问过于频繁 | 超过限流阈值 |

### AutoAuthor 系统配置

AutoAuthor 功能通过 `application.yml` 配置：

```yaml
system:
  auto-author:
    enabled: true  # 是否启用 AutoAuthor 功能
    redis-key-prefix: "autoAuthor:"  # Redis key 前缀
    target-user-id: 1  # 目标用户ID（系统用户）
    # ... 其他配置
```

### Redis 数据结构

#### 1. 扫描进度游标
- **Key**: `autoAuthor:scan:lastId`
- **类型**: String
- **值**: 最后扫描的节点 ID
- **用途**: 记录扫描进度，支持断点续传

#### 2. 就绪队列
- **Key**: `autoAuthor:queue:ready`（具体名称由配置决定）
- **类型**: Set
- **值**: 待处理的节点 ID 集合
- **用途**: 存储等待生成内容的节点

#### 3. AI 会话
- **存储**: 内存中（`AutoAuthorGenerationService`）
- **生命周期**: 进程生命周期或手动重置
- **用途**: 维护与 AI 服务的会话上下文

### 工作流程

```
1. 扫描节点 (手动/定时)
   ↓
2. 加入就绪队列
   ↓
3. 后台任务取出队列中的节点
   ↓
4. 调用 AI 服务生成内容
   ↓
5. 保存生成的帖子
   ↓
6. 更新节点状态
```

### 最佳实践

1. **定期扫描**：
   - 建议配置定时任务自动扫描
   - 手动扫描用于临时补充或测试

2. **手动入队**：
   - 用于优先处理重要节点
   - 用于重试失败的生成任务

3. **会话管理**：
   - 长时间运行后建议重置会话
   - 出现 AI 错误时先尝试重置会话

4. **队列监控**：
   - 定期检查队列长度
   - 队列积压过多时考虑清空重新扫描

### 故障排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 扫描无效果 | 所有节点都已有内容 | 检查目标用户ID配置是否正确 |
| 内容不生成 | AI 服务异常 | 检查 AI 服务配置和网络连接 |
| 队列不消费 | 后台任务未启动 | 检查定时任务配置 |
| 会话错误 | 会话过期或异常 | 调用重置会话接口 |

### 监控指标

建议监控以下指标：
- 队列长度（待处理节点数）
- 扫描进度（当前游标位置）
- 生成成功率
- 生成耗时
- AI 服务调用次数和失败率

```
AdminAutoAuthorController
├── AutoAuthorScanner (扫描服务)
│   ├── NodeDataService (节点数据访问)

```
