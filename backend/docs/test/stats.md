# 统计接口测试文档

> 测试类: `StatsControllerTest`
> 测试目标: 验证统计接口的正确性和数据一致性

---

## 测试环境准备

### 依赖组件
- Redis（用于实时统计数据存储）
- MySQL（用于历史统计数据持久化）
- MockMvc（用于接口测试）

### 测试数据准备
- 创建测试用户
- 创建测试文章
- 准备 Redis 统计数据
- 准备数据库历史统计数据

### 数据清理
- 每个测试前清理 Redis 中的统计数据
- 使用 @Transactional 自动回滚数据库数据

## 三、用户历史统计接口测试

### 3.3 包含今天的数据

## 测试注意事项

1. **Redis 数据清理**
   - 每个测试前清理 `stats:*` 开头的所有 key
   - 避免测试间数据污染

2. **日期处理**
   - 注意时区问题
   - 注意跨天边界情况

3. **数据库事务**
   - 使用 @Transactional 确保测试后自动回滚
   - 避免测试数据残留

4. **异步处理**
   - 同步操作可能涉及异步写入
   - 需要等待异步操作完成再验证

5. **缓存影响**
   - 注意 @Cacheable 对测试的影响
   - 必要时清理缓存
